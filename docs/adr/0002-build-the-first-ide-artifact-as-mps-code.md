# Build the first IDE artifact as MPS code

The first `ide/` artifact will be built as MPS code rather than as a Kotlin IntelliJ Platform plugin. `ide/` will contain a normal MPS project that Gradle builds through an MPS-aware Gradle path, rather than generated MPS sources or hand-managed plugin ZIP assembly. The plugin will add a route to MPS's existing localhost HTTP server instead of owning server lifecycle. Although the endpoint is infrastructure-oriented, prior experience showed that IntelliJ Platform plugin Gradle support is unreliable for MPS plugins, so starting with MPS keeps the build closer to the supported MPS path. Kotlin can still be reconsidered later for narrow pieces if the MPS build can own and package them reliably.
